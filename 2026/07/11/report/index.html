实验一 网络软件基础

1. 实验目标

安装并运行 Cisco Packet Tracer，理解 C/S、B/S、P2P 架构，搭建简单局域网并测试连通性。

2. 实验环境

  • 操作系统：Windows 11
  • 软件：Cisco Packet Tracer 9.0.0.0810
  • 网络要求：IP 段 192.168.56.0/24

3. 实验步骤

3.1 下载与安装

访问 Cisco Networking Academy 官网下载。

3.2 搭建局域网拓扑

使用 “2960” 型号交换机，两台 PC，直通线连接，配置 IP 地址。

3.3 连通性测试

3.4 数据包流动观察

4. 实验结果

成功连通，数据包接收正常。

5. 知识总结

  • C/S、B/S、P2P 架构区别
  • 交换机、PC、直通线连接
  • IP 配置与 ping 测试
  • 仿真模式下观察数据包封装

6. 出现问题

英文界面难以理解，下载中文语言包解决。

7. 心得体会

通过拖拽搭建局域网，直观理解了网络通信的物理与逻辑过程。排错环节（如 IP 不在同一网段）强化了网络故障排查思路。

使用 VirtualBox 安装 Ubuntu Server，配置双网卡，克隆多台虚拟机并实现互通。

2. 实验环境

  • 宿主机操作系统：Windows 11
  • 虚拟化软件：Oracle VirtualBox
  • 镜像文件：Ubuntu Server 22.04 LTS ISO

3. 实验步骤

3.1 创建模板虚拟机

3.2 配置静态 IP 与主机名

3.3 登录系统

3.6 验证多机环境

4. 实验结果

  • NAT / 桥接 / 仅主机三种网络模式

实验七 远程登录管理

1. 实验目标

配置 SSH 服务，实现密钥认证免密登录，使用 SCP 和 rsync 进行文件传输。

2. 实验环境

  • 源虚拟机：node1
  • 目标虚拟机：node2

3. 实验步骤

3.1 安装与检查 SSH 服务

3.2 配置 SSH 服务端

3.3 生成密钥对并复制公钥

3.4 测试免密登录

3.5 使用 SCP 传输文件

3.6 使用 rsync 同步文件

1.rsync同步_推送和拉取

2.同步删除

3.排除指定文件

4. 实验结果

成功在 node1 或 node2 端进行文件推送、拉取、同步删除、排除复制等操作。

5. 知识总结

  • SSH 服务安装与配置
  • 密钥认证（ed25519）生成与分发（ssh-copy-id）
  • 免密登录
  • SCP 和 rsync 文件传输与同步

6. 出现问题

传输方向容易混淆，要明白文件操作的方向。

7. 心得体会

密钥认证比密码认证更安全便捷，适合自动化脚本。rsync 的增量同步和 --delete 参数对备份和部署非常实用，是运维必备工具。

实验八 软件部署

1. 实验目标

综合运用前七项，实现自动化部署（GitHub Actions）和手动部署（Nginx + rsync）。

2. 实验环境

  • 宿主机：node1
  • 云服务器 / 虚拟机：node2
  • GitHub 仓库：私有 git-push，公有 NYeonJ777.github.io

3. 实验步骤

3.1 GitHub Actions 自动化部署

  • 创建源码仓库和部署仓库
  • 关联本地项目并推送
  • 编写工作流文件 deploy.yml
  • 生成 Personal Access Token 并配置 Secret
  • 验证自动部署

3.2 手动部署到虚拟机

  • 安装 Nginx，配置站点目录和虚拟主机

  • 在本地生成静态文件

  • 使用 rsync 同步到服务器
    1.在Node1下载hexo所需工具

    2.Node1上部署博客

    3.gitpush推送

  • 浏览器访问虚拟机 IP 验证

  • 8rsync部署并且访问虚拟机ip以输出结果

4. 实验结果

在外部打开了网页。

5. 知识总结

  • CI/CD 概念
  • GitHub Actions 工作流编写
  • Personal Access Token 配置
  • Nginx 安装与站点配置
  • rsync 部署到云服务器

6. 出现问题

网络连接问题，操作复杂度高。

7. 心得体会

将前七个实验的工具串联起来，完成了从开发到生产部署的完整闭环。自动化构建部署（Actions）和手动 rsync 两种方式对比，加深了对 CI/CD 价值的理解。排错过程锻炼了综合排查能力。
